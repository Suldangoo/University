# 정보 보안의 이해

- 보안의 3대 요소 : 기밀성, 무결성, 가용성
- 기밀성
    - 인가된 사용자만 정보 자산에 접근할 수 있다는 것.
    - 허가되지 않은 사람이 정보에 접근하는 것을 막는 자물쇠
    - 방화벽, 암호, 패스워드 등
- 무결성
    - 적절한 권한을 가진 사용자가 인가한 방법으로만 정보를 변경할 수 있도록 하는 것
    - 예를 들어 지폐의 경우, 정보와 한국은행만이 만들거나 바꿀 수 있음
- 가용성
    - 필요한 시점에 정보 자산에 대한 접근이 가능하도록 하는 것을 의미

- 보안 전문가의 자격 요건
    - 사이버 범죄의 유형
        - 사이버 테러형 범죄는 해커 수준의 범죄, 일반 사이버 범죄는 인터넷을 이용한 일반인 수준의 범죄
        - 해킹이 점점 교묘해져서 추적하기가 어렵기 때문에 검거율이 점점 낮아지는 추세
    - 윤리 의식
        - 윤리 강령
        - 정보통신기반 보호법
        - 클라우드 컴퓨팅법
        - 전자정부법
    - 다양한 분야의 지식
        - 보안 전문가는 다양 분야의 지식을 가지고, 해당 시스템에 대한 지식이 있어야 거기에 보안 지식을 끼얹어 시스템을 구축하는 것임
        - 운영체제, 네트워크, 프로그래밍, 서버, 보안 솔루션, 모니터링, 정책과 절차에 대한 지식이 요구됨

## 시스템 보안의 이해

- 시스템은 **하드웨어**뿐만 아니라 **소프트웨어**까지 매우 많은 거을 포괄
- 시스템 보안 주제
    1. 계정 관리 : 가장 기본적인 인증 수단인 아이디와 패스워드
    2. 세션 관리 : 일정 시간이  지나면 세션 종료로 비인가자의 세션 가로채기 통제
    3. 접근 제어 : 네트워크 안에서 시스템을 다른 시스템으로부터 보호
    4. 권한 관리 : 시스템의 각 사용자가 자신 권한의 정보 자산에 접근
    5. 로그 관리 : 시스템 내부나 네트워크로 외부에서 영향을 미칠 시 내용을 기록하여 관리
    6. 취약점 관리 : 시스템 자체의 결함을 체계적으로 관리
- 개인 리눅스의 경유 user account를 한두개만 생성하면 되지만, 회사나 기업의 경우엔 유저 그룹을 만들어 수많은 유저들을 관리
- 계정관리
    - 식별과 인증 (시스템에 로그인하기 위해 자신이 누군지 알리고 허용하기 위해 확인)
    - 보안의 네 가지 인증 방법
        1. 알고 있는 것 (비밀번호 등 기억중인 정보)
        2. 가지고 있는 것 (신분증, OTP)
        3. 자신의 모습 (홍채,  지문)
        4. 위치하는 곳 (현재 위치, 콜백)
            - 콜백 : 접속 요청 사람의 신원 확인, 등록된 전화번호로 전화를 되걸어 본인 확인, ARS

### 운영체제

- 시스템을 구성하고 운영하는 가장 기본적인 소프트웨어
- 윈도우
    - 윈도우의 관리자 계정은 administrator
    - 관리자 그룹의 계정 존재 형태 확인 명령어 : net localgroup administrators
    - 사용자 계정을 모두 확인 명령어 : net users
    - 시스템에 존재하는 그룹 목록 확인 : net localgroup
- 윈도우의 경우 주요 그룹이 이미 정해져있고, 특징들이 다 정해져있음
- 리눅스의 경우 그룹을 직접 만들고 권한을 커스터마이징할 수 있음
- 유닉스의 계정 관리
    - /etc/passwd 파일에 사용자 계정과 패스워드, 번호, 이름 등이 파일로 저장
    - 유닉스의 그룹 확인 : /etc/group
        - 그룹 이름, 패스워드, 그룹 번호, 계정 목록 등이 출력됨

```bash
groupadd --gid 2000 mgmt
useradd -d /home/mgmt1 -g 2000 -u 2001 -s /bin/sh mgmt1
passwd mgmt1
```

# 네트워크

## 네트워크의 이해

### OSI 7계층

- 국제표준화기구(ISO)는 다양한 네트워크간 호환을 위해 만든 표준 네트워크 모델
- **물**리 - **데**이터링크 - **네**트워크 - **전**송 - **세**션 - **표**현 - **응**용
- 물리 계층 (1계층)
    - 시스템간의 연결을 의미하는 것
    - 물리적인 케이블 연결을 담당하고, 쓰이는 케이블을 데이터 속도에 따라 분류
        - CAT1 ~ CAT7
        - 한국은 보통 CAT5 ~ CAT6 케이블을 쓰며 기가인터넷은 CAT7 사용
    - 케이블 선은 일반적으로 UTP를 사용
    - 랜 케이블의 연결 커넥터는 RJ45
- 데이터 링크 계층 (2계층)
    - 두 포인트 간의 신뢰성있는 전송을 보장하는 계층
    - MAC 주소 (물리적 주소)
        - 데이터 링크 계층에서 상호 통신을 위해 할당받는 주소
        - 총 12개의 16진수로 구성
            - 앞 6자리는 OUI 코드 (이 코드를 알면 제조사를 알 수 있다)
            - 뒤 6자리는 Host Identifier
        - 스위치 장비를 이용해 패킷의 흐름을 제어
            - 스위치의 여러 포트에 각각 장비가 연결되어있으며, 장비의 MAC주소가 저장됨
- 네트워크 계층 (3계층)
    - 여러 개의 노드를 거칠 때마다 경로를 찾아주는 역할
    - 전달 과정에서 라우팅, 흐름 제어, 세그먼테이션, 오류 제어 등을 수행
    - IP 주소
        - 네트워크 계층에서 여러 개의 노드를 거쳐 경로를 찾기 위한 주소
        - 8비트의 수 4개로 이루어져있음 (4byte)
            - 192.168.0.1
        - IP 주소의 클래스는 5개로 구분해놓고 있음 (A, B, C, D, E)
        - A클래스는 0으로 시작하여, 첫 번째 자리가 네트워크 주소
        - B클래스는 10으로 시작하여, 두 번째 자리까지 네트워크 주소…
    - 네트워크 계층에서 사용하는 대표적인 네트워크 장비는 라우터
        - 스위치에서 온 패킷을 라우터로 전송, 라우터가 인터넷으로 패킷 전송
    - IP 주소가 부족한 대표적인 이유는 IoT. 그래서 IPv6를 만듦
- 전송 계층 (4계층)
    - 양 끝단의 사용자들이 신뢰성 있는 데이터를 주고받을 수 있게 함
    - 유명한 전송 프로토콜인 TCP, UDP (포트 주소)
    - 출발지 포트 결정
        - 보통 1,025 ~ 65,535번 중에서 사용하지 않는 임의의 포트를 사용
    - TCP의 경우 두 시스템이 항상 연결중인지 확인을 하고 있음
    - TCP는 3-웨이 핸드셰이킹으로 연결
- 세션 계층 (5계층)
    - 양 끝단의 으용 프로세스가 통신을 관리하기 위한 방법을 제공
    - 동시 송수신, 반이중, 전이중 방식의 통신
- 표현 계층 (6계층)
    - 코드 간의 번역을 담당
    - ASN.1 방식
- 응용 프로그램 계층 (7계층)
    - 사용자나 응용 프로그램 사이에 데이터 교환이 가능하게 하는 계층
    - HTTP, FTP 등의 서비스 제공

## 서비스 거부 공격 (DoS, DDoS)

### DoS

- 다른 해킹에 비해 비교적 간단한 공격, 일종의 훼방
- 취약점 공격형
    - 특정 형태의 오류가 있는 네트워크 패킷 처리 로직에 공격
    - 보잉크 / 봉크 / 티어드롭 공격, 랜드 공격
- 자원 고갈 공격형
    - 네트워크 대역폭이나 시스템, CPU 자원을 소모시키는 형태
    - 랜드 공격, 죽음의 핑, SYN 플러딩, 스머프 공격, 메일 폭탄 공격
- 보잉크 / 봉크 / 티어드롭 공격
    - 프로토콜 오류 제어 로직을 악용, 시스템 자원 고갈
    - TCP는 데이터 전송 시 신뢰 확보용으로 패킷에 문제가 있으면 반복적으로 재요청
        - 패킷의 순서가 올바른지 확인, 손실된 패킷이 있다면 재전송을 요구
    - 해당 공격으로 공격 대상이 **반복적인 재요청과 수정을 계속**하게 함, **자원 고갈**
- 랜드 공격
    - land는 나쁜 상태에 빠지게 하다라는 뜻
    - 패킷 전송 시 **출발지 IP 주소와 목적지 IP 주소를 똑같이** 만들어 공격 대상에게 전송
    - 동시 사용자 수를 점유하고 CPU 부하를 올려 시스템이 지침
- 죽음의 핑 공격
    - 네트워크는 연결 상태를 점검하는 ping 명령을 보낼 때 공격 대상에게 최대한 패킷을 쪼갬
    - 공격 대상 시스템은 대량의 작은 패킷을 수신하느라 네트워크가 마비
- SYN 플러딩 공격
    - 네트워크에서 시스템의 동시 사용자 수 제한을 이용한 공격
    - 존재하지 않는 클라이언트가 접속 가능 공간에 접속한 것처럼 속여 다른 사용자를 제한
    - TCP의 연결 과정인 3-웨이 핸드셰이킹의 문제점을 악용
    - 특정 웹 서버의 접속자가 폭주하여 서버 접속이 되지 않고, 마비됨
- HTTP GET 플러딩 공격
    - HTTP의 GET 메소드를 무한대로 실행하여 시스템 과부하
- HTTP CC 공격, 동적 HTTP 리퀘스트 플러딩 공격, 슬로 HTTP 헤더 DoS 공격, 슬로 HTTP POST 공격 등등… 모두 HTTP의 메소드를 지속적으로 요청해 자원을 고갈시켜 서비스를 방해하는 공격
- 스머프 공격
    - ICMP 패킷과 네트워크에 존재하는 시스템을 이용, 패킷 확장으로 서비스 거부 공격 수행
    - 다이렉트 브로드캐스트를 악용하여 공격 방법이 간단
    - 공격을 수행한 후 다른 정직한 패킷에게 책임을 넘겨 정직한 패킷이 범인처럼 보이게 오해
    - 공격자가 공격 대상에게 위조된 ICMP reply를 보내 받은 공격 대상은 시스템 과부하가 옴
- 메일 폭탄 공격
    - 스팸 메일 잔뜩 보내는 공격

### DDOS

- 분산 서비스 거부 공격
- 아직까지 확실한 대책이 없으며 공격자 위치와 구체적인 발원지를 파악하는 것이 거의 불가능
- 공격자가 여러 마스터들을 사용해 분산된 컴퓨터들로 데몬 프로그램을 실행, 공격 대상을 공격
    - 다른 컴퓨터들에게 데몬 프로그램을 깔면 좀비 PC가 되고, 공격을 수행하게 됨
    - 좀비 PC가 된 컴퓨터들은 피해자인 동시에 공격자
    - 좀비 PC끼리 형성된 네트워크를 봇넷이라고 함
    - 공격자가 명령을 내리거나 정해진 스케줄에 따라 일제히 공격을 하게 됨

## 스니핑 공격

- 스니핑 : 코를 킁킁거린다는 효과음
- 공격할 때 아무것도 하지 않고 조용히 데이터 속에서 정보만을 찾는 것 (수동적 공격)
    - 다른 사람의 대화를 도청
    - 전화선이나 UTP에 태핑을 하여 전기적 신호를 분석, 정보 탐색
    - 전기적 신호를 템페스트 장비로 분석
- 원리
    - 네트워크 카드가 패킷의 IP주소와 MAC주소를 인식, 저장할지 결정
    - 인식된 정보가 자신의 것과 일치하지 않으면 해당 패킷을 무시
    - **프러미스큐어스** : 네트워크 필터링 해제 상태
    - 프러미스큐어스를 통해 그냥 모든 패킷을 죄다 받고 데이터를 분석
- 공격의 종류
    - 스위치 재밍 공격 : 스위치 장비 하드웨어적 공격
    - SPAN 포트 태핑 공격 : 스위치 포트 미러링 기능 이용, 하드웨어 장비를 이용한 공격
- 스니핑 공격의 탐지
    - 스니퍼를 설치하면 이상 현상을 일으키지 않기 때문에 사용자가 인지하기 어려움
    - 쉽게 탐지하려면 스니퍼가 프러미스큐어스 모드에서 작동하지 않는다는 점을 이용
    - 예를 들어, 스니퍼를 탐지할 땐 출석을 불렀는데 두 명이 동시에 대답한걸 발견

## 스니핑 공격

- 스니핑 공격의 탐지
    - 스니퍼를 설치하면 이상 현상을 일으키지 않기 때문에 사용자가 인지하기 어려움
    - 쉽게 탐지하려면 스니퍼가 프러미스큐어스 모드에서 작동하지 않는다는 점을 이용
    - 예를 들어, 스니퍼를 탐지할 땐 출석을 불렀는데 두 명이 동시에 대답한걸 발견
- ping을 이용한 스니퍼 탐지
    - 대부분 스니퍼는 TCP/IP에서 동작하기 때문에 request를 받으면 response를 전달
    - 의심이  가는 호스트에 ping을 보내 스니퍼를 탐지
    - 이 때 위조단, 존재하지 않는 MAC주소를 위장해서 전송하고 reply를 받으면 스니퍼
    - 프로미스큐어스 모드기 때문에 위조된 ping도 받고 ICMP echo reply를 주는 것
- ARP를 이용한 스니퍼 탐지
    - 위조된 ARP를 보냈을 때 response 오면 프러미스큐어스 모드인 것
- DNS를 이용한 스니퍼 탐지
    - DNS lookup을 수행하여 없는 DNS를 받아 IP 던지면 프러미스큐어스 모드인 것
- 유인을 이용한 스니퍼 탐지
    - 공격자의 주요 목적은 ID, PW 획득. 이 점을 이용해 가짜 ID와 PW를 뿌리고 이 ID와 PW로 로그인 시도를 감지하면 스니퍼 탐지
- ARP watch를 이용한 스니퍼 탐지
    - ARP watch : MAC주소와 IP주소의 매칭 값을 초기에 저장, ARP 트래픽을 모니터링하여 이를 변하게 하는 패킷이 탐지되면 관리자에게 메일로 알려주는 툴
    - 대부분 공격 기법은 위조된 ARP를 사용하기 때문에 쉽게 탐지할 수 있음

# 스푸핑

## ARP 스푸핑 공격

- ARP 스푸핑의 개요
    - ARP 스푸핑 : MAC 주소를 속이는 것
    - 로컬에서 통신하는 서버와 클라이언트의 IP주소에 대한 MAC 주소를 공격자의 MAC주소로 속임. 클라이언트 ↔ 서버 패킷이 공격자에게 향하게 하여 랜의 통신 흐름을 왜곡
    - 양 컴퓨터에게 목적지의 IP(진짜 IP)가 위조된 MAC주소(공격자 주소)를 보내는 것
- 삼자 사기같은 느낌. 서로가 보낼 패킷 사이에 끼어든 원리

## IP 스푸핑 공격

- IP 스푸핑의 개요
    - IP주소를 속이는 것. 다른 사용자의 IP를 강탈하여 권한을 획득
    - 트러스트(신뢰)를 맺고 있는 서버와 클라이언트 확인, 클라이언트에 서비스 거부 공격을 하여 연결을 끊음. 그 클라이언트의 IP 주소를 확보하여 실제 클라이언트처럼 패스워드 없이 접근
    - 트러스트 : 클라이언트에 신뢰되는 정보를 서버에 저장하고, 서버에 접근하면 패스워드 없이 로그인을 허락하는 인증법. 유닉스에선 트러스트, 윈도우에선 액티브 디렉터리를 사용
        - 유닉스에선 /etc/host.equiv 파일에 접속 가능한 IP와 아이디 등록
- IP 스푸핑의 서버 접근
    - 스니핑으로 패스워드를 알아낼 필요가 없음.
    - 공격 대상에게 서비스 거부 공격(Dos)을 수행하여 해당 IP의 출연을 막음
    - 공격자는 자신이 해당 IP로 설정 변경 후 서버에 접속하는 형태로 진입
- ICMP 리다이렉트
    - ICMP 리다이렉트를 이용하면 공격자 스스로가 라우터가 됨
    - ICMP 리다이렉트 패킷도 공격 대상에게 보낸 후 라우터 A에 연결하면 모든 패킷 스니핑

## DNS 스푸핑

- 실제 DNS 서버보다 빨리 공격 대상에게 DNS response 패키을 보내 공격 대상이 잘못된 IP 주소로 웹 접속을 하도록 유도하는 공격
- 가짜 웹 주소에 접속하는 공격. 링크를 통해 들어갔는데 쇼핑몰이나 포르노 사이트가 뜨는 경우
- 원본 웹과 똑같은 웹을 만들어 공격자에게 전달, ID와 PW를 입력하면 그대로 받아옴
- DNS 스푸핑 공격을 막으려면 중요 서버에 DNS query를 보내지 않으면 됨
    - hosts 파일에 등록된 정보를 확인, URL과 IP 정보가 등록

## 무선 랜

- 무선 랜의 개요
    - 유선 랜의 네트워크를 확장하려는 목적
    - 이를 위해서 내부의 유선 네트워크에 AP(Access Point)장비를 설치해야 함
    - 확장된 무선 네트워크는 AP를 설치한 위치에 따라 통신 영역 결정
    - AP와 허브 - 라우터 - 인터넷으로 연결
    - 안테나의 형태에 따라 방향과 길이가 달라짐
        - 무지향성 안테나 : 봉의 형태, 수신에 일정한 방향이 없음.
        - 지향성 안테나 : 접시 혹은 쟁반 모양, 수직과 수평으로 나뉘고, 목표 방향이 있으며, 해당 방향으론 통신 거리가 더 긴 편.
- AP 보안
    - 물리적인 보안 및 관리자 패스워드 변경
    - SSID 브로드캐스팅 금지
        - SSID : 무선 랜 네트워크 검색 시 확인할 수 있는 AP 목록 중 이름으로 표시된 것
        - 높은 수준의 보안 권한이 필요한 무선 랜은 SSID 브로드캐스팅을 차단
- 무선 랜 통신의 암호화
    - 무선 랜 통신 과정 데이터 유출을 막는 것뿐 아니라 네트워크 인증
    - 암호화된 통신을 수행하려면 패스워드를 입력해야 함
- WEP
    - 무선 랜 통신을 암호화하기 위해 802.11b 프로토콜부터 적용
    - RC 4 암호화 알고리즘을 기본으로 사용 → 이거 현대에는 깨짐
    - 24비트의 IV를 사용하는데, 이  IV를 충분히 수집하여 크래킹하면 1분만에 복호화 가능
- WPA-PSK
    - 더 개선된 802.11i 보안 표준의 일부분, WEP 방식의 보안 문제점 개선
    - WPA-1, WPA-2 규격이 포함되어 있음
    - TKIP가 RC 4 알고리즘을 암호에 사용하는 반면, CCMP는 AES 알고리즘을 기반으로 함
- EAP와 802.1x 암호화
    - 위 방식과 차이점은 아이디와 패스워드를 통한 사용자 인증
    - 무선 랜 연결 (세션) 별로 재사용이 불가능한 다른 암호화 키 사용
    - 기존 방식은 무선 랜 사용자 ↔ AP였지만, 이 방식은 거기에 인증용 RADIUS 서버가 또 연결되어있음. 해당 방식을 크래킹하려면 폐쇄망인 해당 서버에 접근해야해서 더 강함
